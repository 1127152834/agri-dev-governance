# Task Pool Manager

## Overview

在项目层面统一管理任务池中的真实任务数据，确保任务从发布、指派、开发、测试、评审到归档都在同一套目录和日志结构中留痕。

本 Skill 不初始化成员资料，也不代替具体开发实现。

## Responsibilities

- 初始化任务池目录
- 发布任务到任务池
- 更新任务状态
- 写入任务流转日志
- 维护任务索引

## Required Inputs

至少收集以下信息：

- 任务编号
- 任务标题
- 所属模块
- 当前动作
- 执行人
- 说明

## Lifecycle

需要详细状态说明时，读取 `references/lifecycle-states.md`。

## Rules

- 不要因为认领、指派或完成而删除任务文件
- 所有状态流转都必须写入日志
- 任务池是任务主记录，成员目录只做个人视角索引

## Output Requirements

输出必须明确包含：

- 更新了哪个任务文件
- 更新到了哪个状态目录
- 是否更新了索引和日志
