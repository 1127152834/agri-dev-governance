# Task Claim

## Overview

在开发人员认领任务或接受指派任务后，使用本 Skill 将任务池状态和成员侧状态同步起来，确保个人任务面板、状态面板和模块文档入口保持一致。

本 Skill 不负责发布任务，也不直接执行开发实现。

## Required Inputs

至少收集以下信息：

- 成员名
- 任务编号
- 模块名
- 任务标题
- 当前动作

## Sync Actions

认领后至少完成以下同步动作：

- 将任务从 `01-ready` 更新到 `02-assigned`
- 在任务日志中新增认领记录
- 更新成员 `tasks.md`
- 更新成员 `status.md`
- 创建模块目录入口

需要细化同步范围时，读取 `references/member-sync.md`。

## Rules

- 不要跳过任务池状态更新直接只改成员目录
- 不要在没有成员初始化时执行认领同步
- 不要把成员目录当作任务真相源

## Output Requirements

输出必须明确包含：

- 任务是否已进入 `02-assigned`
- 更新了哪些成员侧文件
- 是否已建立模块目录入口
