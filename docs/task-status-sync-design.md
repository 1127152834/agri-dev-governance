# task-status-sync 设计文档

## 1. 目标

为团队提供统一的任务状态同步 Skill，用于在任务池、成员任务面板、成员状态面板和模块文档状态不一致时，执行状态对齐与差异修正。

## 2. 边界

`task-status-sync` 只负责状态同步与差异修正，不负责任务创建、成员初始化或任务认领本身。

它负责：

- 对齐任务池与成员侧状态
- 更新成员 `tasks.md`
- 更新成员 `status.md`
- 对齐模块文档目录的阶段状态

它不负责：

- 发布任务
- 认领任务
- 直接执行开发实现

## 3. 主记录原则

任务池中的任务文件始终是主记录。

成员目录中的：

- `tasks.md`
- `status.md`
- 模块文档目录

都属于派生视图或配套视图，发生冲突时默认以任务池状态为准。

## 4. 输入

- 任务编号
- 当前任务池状态
- 当前成员目录状态
- 当前模块目录状态
- 同步动作或目标状态

## 5. 输出

- 同步后的成员 `tasks.md`
- 同步后的成员 `status.md`
- 同步后的模块目录状态
- 冲突说明或修正记录

## 6. 同步目标

- `docs/task-pool/`
- `docs/成员名/tasks.md`
- `docs/成员名/status.md`
- `docs/成员名/模块名称/`

## 7. 强制原则

- 冲突时优先以任务池为准
- 同步动作必须明确说明修改了哪些视图
- 如果无法确定正确状态，必须先报告冲突，不允许擅自猜测
