# 任务认领规范

## 1. 定位

任务认领规范用于定义开发人员如何从任务池中认领任务，以及认领后如何同步到成员侧目录。

## 2. 前置条件

认领前必须满足：

- 成员已完成 `member-init`
- 任务已处于 `01-ready`
- 任务池中存在完整任务记录

## 3. 认领动作

认领时至少要完成以下动作：

1. 将任务状态从 `01-ready` 更新为 `02-assigned`
2. 在 `task-ledger.md` 中新增认领日志
3. 将任务写入成员 `tasks.md` 的“未开始”
4. 更新成员 `status.md` 的当前主任务
5. 创建模块目录入口

## 4. 模块目录入口

认领后应在成员目录下建立模块文档入口，至少保证以下路径存在：

```text
docs/
└── 成员名/
    └── 模块名称/
        ├── 01-需求说明.md
        ├── 02-产品设计.md
        ├── 03-实施计划.md
        ├── 04-开发记录.md
        ├── 05-测试报告.md
        ├── 06-发布说明.md
        └── 07-复盘总结.md
```

首版可以先建立空文件或模板入口，后续由对应 Skill 填充内容。

## 5. 同步原则

- 任务池记录是主记录
- 成员 `tasks.md` 和 `status.md` 仅为同步视图
- 任一同步动作失败时，必须明确失败项
