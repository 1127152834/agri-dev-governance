# 日报规范

## 1. 定位

日报规范用于定义成员个人日报的生成路径、字段结构和使用方式。

## 2. 输出路径

推荐输出路径：

```text
docs/成员名/reports/daily/2026/2026-04-01.md
```

## 3. 内容结构

日报至少包含：

- 日期
- 今日完成
- 进行中
- 阻塞项
- 明日计划
- 关联任务 / 模块

## 4. 数据来源

日报草稿可优先读取：

- `status.md`
- `tasks.md`

然后由成员补充阻塞项、明日计划和补充说明。

## 5. 使用原则

- 每日生成一份
- 同一天的日报优先更新已有文件，不重复创建多个版本
- 草稿生成后，应由成员确认内容是否准确
