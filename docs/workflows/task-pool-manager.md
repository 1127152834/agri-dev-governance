# 任务池管理规范

## 1. 定位

任务池管理规范用于统一项目任务的发布、指派、流转与归档方式。项目经理或任务管理角色应通过统一的任务池目录来维护任务，不允许把任务记录散落在成员目录中。

## 2. 任务池目录

```text
docs/
└── task-pool/
    ├── 00-draft/
    ├── 01-ready/
    ├── 02-assigned/
    ├── 03-in-progress/
    ├── 04-waiting-test/
    ├── 05-waiting-review/
    ├── 06-done/
    ├── 07-blocked/
    ├── 08-cancelled/
    ├── 99-archived/
    ├── task-index.md
    └── task-ledger.md
```

## 3. 核心规则

- 任务从任务池产生
- 任务认领或指派后，不删除任务文件，只改变状态目录
- 所有状态变化都必须追加日志
- 成员目录中的 `tasks.md` 只作为个人视角索引，不是任务真相源

## 4. 任务发布

发布任务时至少要写清：

- 任务编号
- 任务标题
- 所属模块
- 优先级
- 验收标准
- 前置依赖
- 当前状态

## 5. 任务指派与认领

任务进入 `01-ready/` 后，可被认领或被项目经理直接指派。

一旦认领或指派：

- 任务进入 `02-assigned/`
- `task-ledger.md` 必须新增记录
- 成员 `tasks.md` 与 `status.md` 应同步更新

## 6. 任务归档

已完成、已取消或历史任务最终进入 `99-archived/`。

归档前必须保留完整流转日志，不允许直接删除任务文件。
