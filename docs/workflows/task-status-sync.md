# 任务状态同步规范

## 1. 定位

任务状态同步规范用于定义任务状态在任务池、成员任务面板、成员状态面板和模块文档入口之间如何保持一致。

## 2. 主记录

主记录永远是任务池中的任务文件及其所在状态目录。

以下内容属于派生视图：

- 成员 `tasks.md`
- 成员 `status.md`
- 模块文档目录

## 3. 何时需要同步

以下场景应触发同步：

- 任务认领后
- 任务状态从一个阶段流转到另一个阶段后
- 任务阻塞、取消、完成后
- 成员视图与任务池视图不一致时

## 4. 同步范围

至少同步以下位置：

- `docs/task-pool/`
- `docs/成员名/tasks.md`
- `docs/成员名/status.md`
- `docs/成员名/模块名称/`

## 5. 冲突处理

### 5.1 默认原则

以任务池状态为准。

### 5.2 冲突示例

- 任务池显示 `03-in-progress`，成员 `tasks.md` 仍在“未开始”
- 任务池显示 `05-waiting-review`，但成员 `status.md` 仍写“开发中”
- 任务池显示 `08-cancelled`，成员模块目录仍被当作进行中任务维护

### 5.3 处理规则

- 明确列出不一致项
- 按任务池状态修正成员视图
- 若任务池本身也不可信，则先停止同步并报告冲突

## 6. 禁止事项

- 禁止在不确认主记录的情况下擅自猜测状态
- 禁止跳过任务池直接修改成员视图并视为完成同步
