# task-claim 设计文档

## 1. 目标

为开发人员提供统一的任务认领入口，在任务被认领或指派后，将项目侧任务池状态同步到成员侧任务面板、状态面板和模块文档目录入口。

## 2. 边界

`task-claim` 只负责认领后的成员侧同步，不负责发布任务和管理任务池本身。

它负责：

- 将任务从 `01-ready` 进入 `02-assigned`
- 更新任务池日志
- 更新成员 `tasks.md`
- 更新成员 `status.md`
- 初始化模块文档目录入口

它不负责：

- 创建任务池任务
- 替代产品设计或开发实现
- 初始化成员基础目录

## 3. 输入

- 成员名
- 任务编号
- 模块名
- 任务标题
- 当前动作（认领 / 接受指派）

## 4. 输出

- 任务状态进入 `02-assigned`
- 成员 `tasks.md` 更新到“未开始”
- 成员 `status.md` 更新当前主任务
- 模块目录入口创建完成

## 5. 同步目标

默认同步以下位置：

- `docs/task-pool/02-assigned/`
- `docs/成员名/tasks.md`
- `docs/成员名/status.md`
- `docs/成员名/模块名称/`

## 6. 强制原则

- 认领动作必须写入任务日志
- 成员侧同步失败时，不应宣称认领完成
- 任务详情仍以任务池文件为主记录
