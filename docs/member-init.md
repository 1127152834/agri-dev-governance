# 成员初始化规范

## 1. 定位

成员初始化规范用于统一团队成员的个人资料、状态面板、任务容器和日报归档结构。所有开发成员首次加入项目时，都应先完成本规范初始化。

## 2. 初始化目录

```text
docs/
└── 成员名/
    ├── profile.md
    ├── status.md
    ├── tasks.md
    ├── tasks/
    └── reports/
        └── daily/
            └── README.md
```

## 3. 文件职责

### 3.1 `profile.md`

用于记录成员静态信息，包括：

- 姓名
- 角色
- Git 用户名
- 负责仓库
- 负责模块
- 技能方向
- 协作说明

### 3.2 `status.md`

用于记录成员当前工作状态，包括：

- 当前主任务
- 当前仓库
- 当前分支
- 当前 worktree
- 阻塞项
- 待协作项
- 最近更新时间

### 3.3 `tasks.md`

用于记录成员的任务总览，预置状态栏目：

- 未开始
- 进行中
- 待测试
- 待评审
- 已完成
- 已阻塞

### 3.4 `tasks/`

用于存放单任务详细记录文件。`member-init` 只初始化目录，不创建真实任务内容。

### 3.5 `reports/daily/README.md`

用于定义日报命名规则、字段结构和归档方式。

## 4. 日报规则

推荐按年份归档：

```text
reports/daily/2026/2026-03-31.md
```

首版初始化时只建立 `reports/daily/` 和说明文件，后续日报由专门的日报流程生成。

## 5. 使用要求

- 每个成员仅初始化一次主目录
- 目录名统一使用成员姓名
- 后续任务和日报必须落入该目录结构，不允许自定义散落路径
