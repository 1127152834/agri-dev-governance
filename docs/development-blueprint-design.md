# 开发治理蓝图页面设计文档

## 1. 目标

为治理仓库提供一份面向开发者的可视化开发治理总装图，帮助团队成员快速理解任务从提出、拆分、认领、设计、开发、测试、评审到归档的完整闭环，同时明确任务取消、阻塞、需求变更、测试不通过、评审驳回和功能重做等异常回流路径。

## 2. 设计原则

- 页面面向开发者阅读，优先表达流程清晰度，而不是装饰性。
- 采用工程蓝图风格，传达专业、精确、可信赖的技术治理氛围。
- 视觉风格与文案分离，页面文案保持中文。
- 结构化保存流程数据，避免把流程规则硬编码进绘图逻辑。
- 主链路优先可读，异常链路在主图中可见但不抢主流程注意力。

## 3. 页面结构

页面由四个区域组成：

1. 顶部图框标题栏：显示图名、版本、日期、适用范围与阅读提示。
2. 中央主画布：使用 Canvas 绘制开发治理闭环总图。
3. 右侧详情面板：点击节点后展示阶段说明、责任角色、输入、输出、门禁与异常处理。
4. 底部图例与控制区：展示主流程、异常流程、终态节点的色彩语义，并提供主流程 / 全流程切换。

## 4. 主流程节点

主流程按以下顺序排列：

1. 任务提出
2. 任务拆分
3. 任务发布
4. 认领 / 指派
5. 需求说明
6. 产品设计
7. 实施计划
8. 创建 worktree
9. 开发实现
10. 测试验证
11. 提交 PR
12. 评审处理
13. 合并归档

## 5. 异常流程

页面必须展示以下异常节点与回流关系：

- 任务取消：从任务发布、认领 / 指派、需求说明阶段进入取消归档。
- 任务阻塞：从任意执行阶段可进入阻塞，解除后回原阶段。
- 需求变更：从开发实现、测试验证、评审处理回退到需求说明或实施计划。
- 测试不通过：从测试验证回退到开发实现。
- 评审驳回：从评审处理回退到开发实现。
- 功能重做：从评审处理或合并后的复盘场景回退到任务拆分或产品设计。

## 6. 交互设计

- Hover 节点：节点描边高亮，相关连线同步高亮。
- Click 节点：在右侧展示节点详细说明。
- 切换视图：支持切换为“仅主流程”或“主流程 + 异常分支”。
- 自适应缩放：画布根据容器尺寸自动重绘。

## 7. 文件结构

- `docs/development-lifecycle.html`：页面结构
- `docs/development-lifecycle.css`：蓝图风格与布局样式
- `docs/development-lifecycle-data.js`：流程节点、异常关系与图例数据
- `docs/development-lifecycle-canvas.js`：Canvas 绘图、命中检测与交互逻辑

## 8. 验证方式

- 静态打开页面可正常显示
- 节点悬停与点击交互生效
- 主流程切换与异常流程显示正确
- 页面在桌面常见宽度下可读性良好
