# team-dev-spec 实施计划

## 1. 目标

为治理仓库补齐一份团队开发总规范文档与对应的 `team-dev-spec` Skill，作为开发前的统一门禁入口。

## 2. 任务拆解

### 任务一：补齐设计文档

- 说明 `team-dev-spec` 的定位、职责边界、输入输出和依赖文档。

### 任务二：编写规范文档

- 新建 `docs/workflows/team-dev-spec.md`
- 定义适用范围、门禁规则、阶段判断、分流映射和禁止事项。

### 任务三：编写 Skill

- 完成 `skills/team-dev-spec/SKILL.md`
- 保持内容简洁，重点表达触发条件、检查动作与分流规则。

### 任务四：补齐引用资源

- 如有必要，补参考文件以容纳阶段分流细节，避免 `SKILL.md` 过胖。

### 任务五：验证与提交

- 运行 skill 校验脚本
- 自查文档与 Skill 是否一致
- 提交当前改动
