# 开发治理蓝图页面实施计划

## 1. 目标

交付一份基于 HTML + Canvas 的开发治理总装图页面，完整呈现开发闭环与异常回流，并满足轻交互浏览需求。

## 2. 任务拆解

### 任务一：补齐流程设计文档

- 新建设计文档，明确页面结构、节点清单、异常回流与交互边界。

### 任务二：建立页面骨架

- 创建 HTML 结构，放置标题栏、主画布、详情面板、图例与开关区域。
- 预留适合后续脚本挂载的数据节点和容器。

### 任务三：建立蓝图视觉系统

- 创建蓝图风格 CSS，包括深蓝背景、主次网格、图框、技术字体、图例配色和交互高亮样式。

### 任务四：定义流程数据

- 使用结构化数据描述分组、节点、主流程边、异常边和默认选中项。
- 避免绘图逻辑和流程规则耦合。

### 任务五：实现 Canvas 绘图与交互

- 绘制分组框、节点、连线、箭头、异常回流和装饰性标尺元素。
- 实现 hover 命中检测、节点高亮、点击展示详情、视图切换与重绘逻辑。

### 任务六：本地验证

- 使用本地静态服务器打开页面进行交互验证。
- 确认节点高亮、详情面板、流程切换与布局表现符合预期。

## 3. 交付物

- `docs/visuals/development-lifecycle/index.html`
- `docs/visuals/development-lifecycle/styles.css`

- 本实施计划与设计文档

## 4. 风险控制

- 如果 Mermaid 或纯 DOM 布局不能满足视觉和交互要求，优先使用 Canvas 实现。
- 如果异常流过多导致画面拥挤，允许通过视图切换降低默认复杂度，但不能丢失异常闭环定义。
- 如果样式和脚本耦合过高，优先回退为数据驱动的绘制结构。

## 5. X6 重构计划

### 任务七：替换为 X6 图引擎

- 使用 X6 重建节点、分组、连线和基础交互。
- 保留数据驱动结构，不把流程信息写死在视图代码里。

### 任务八：重构页面结构与样式

- 将当前主图区域改造成 X6 容器。
- 保留蓝图风格，但重点优化图面布局、呼吸感和连线清晰度。

### 任务九：回归验证

- 重新验证节点点击、详情联动、主流程视图切换与页面加载稳定性。

### 任务十：交互体验优化

- 打开画布拖拽平移能力。
- 消除不必要的整图重绘，减少卡顿。

### 任务十一：节点提示词体系

- 为每个主流程节点补充推荐 Skill。
- 为每个节点补充标准参考提示词模板。
- 在详情面板中新增“推荐 Skill”和“参考提示词”区域。
