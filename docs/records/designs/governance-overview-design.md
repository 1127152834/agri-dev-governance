# governance-overview 设计文档

## 1. 目标

为治理仓库提供一份总览文档，把当前已经建设好的开发治理能力、Skill 调用顺序、角色分工、目录结构和执行闭环统一说明，帮助团队成员快速理解并落地使用。

## 2. 定位

`governance-overview` 不是新的 Skill，而是一份组织级索引文档。

它负责：

- 解释当前治理体系由哪些能力组成
- 说明推荐的调用顺序
- 说明哪些内容已并入主分支，哪些仍待合并
- 统一角色与目录认知

## 3. 内容范围

- 当前治理地图
- 角色分工
- Skill 调用顺序
- 文档与模板目录
- 当前仓库现状
- 后续建议

## 4. 风险控制

- 不夸大尚未并入主分支的内容
- 不把规则正文重复一遍，优先做索引和串联
- 只引用已存在的文档与 Skill
