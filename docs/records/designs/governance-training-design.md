# 治理体系培训 Deck 设计文档

## 1. 目标

为智慧农业团队制作一份 Web Slides 培训材料，用于在 40 分钟左右的培训中，帮助项目经理、开发人员和使用 Codex 的成员理解并掌握当前治理仓库的整体结构和实际使用方式。

## 2. 形式

- 单文件 HTML 演示稿
- 零依赖
- 浏览器直接打开即可播放
- 适合内部培训与屏幕共享

## 3. 讲法

采用混合型结构：

- 前半段建立治理认知
- 中段用一个完整场景串联 Skill
- 后半段收束常见错误和落地建议

## 4. 培训时长

目标时长：40 分钟左右。

## 5. 目标受众

- 项目经理
- 开发人员
- AI 助手 / 使用 Codex 的成员

## 6. 视觉方向

参考 `frontend-slides` 的单文件 HTML 思路，采用：

- 深色底
- 专业、克制、偏治理和架构感
- 动画适度，突出层次，不做花哨炫技

## 7. 页纲

1. 封面
2. 为什么要做治理
3. 治理体系总览
4. 完整闭环流程
5. 角色分工
6. 演示场景介绍
7. 项目初始化
8. 成员加入
9. 任务池发布与指派
10. 开发人员认领任务
11. 开发前门禁
12. 提交与交付
13. 状态同步与每日日报
14. 常见错误与落地建议

## 8. 演示主线

统一示例场景：

`智慧农业平台 -> 设备管理模块 -> 从项目初始化到日报收尾`

## 9. 风险控制

- 每页必须严格控制信息密度，不允许滚动
- 不把所有规范原文搬进 slides，只保留培训需要的内容
- 统一以当前主分支中的文档和 Skill 为依据
