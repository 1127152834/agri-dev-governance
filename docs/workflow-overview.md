# 开发流程总览

可视化版本请查看：`docs/development-lifecycle.html`。

## 主流程

```text
成员初始化
→ 需求登记
→ 产品设计
→ 实施计划
→ 创建 worktree
→ 开发实现
→ 测试验证
→ 提交与 PR
→ 评审合并
→ 文档回写与复盘
```

## 阶段说明

### 1. 成员初始化

- 输入：成员加入团队或首次参与项目
- 输出：个人资料、任务列表、状态面板
- 门槛：成员目录与基础信息初始化完成

### 2. 需求登记

- 输入：新需求、缺陷、优化项
- 输出：需求说明文档
- 门槛：需求范围、目标与责任人明确

### 3. 产品设计

- 输入：需求说明
- 输出：产品设计文档，必要时补充 UI / 交互设计
- 门槛：核心流程、边界与成功标准明确

### 4. 实施计划

- 输入：产品设计文档
- 输出：实施计划文档
- 门槛：任务拆解、影响范围与验证方式明确

### 5. 创建 worktree

- 输入：实施计划、任务编号、目标分支
- 输出：独立工作区与开发分支
- 门槛：分支命名与工作区命名符合规范

### 6. 开发实现

- 输入：实施计划、设计文档、独立工作区
- 输出：代码改动、开发记录
- 门槛：实现范围与计划保持一致

### 7. 测试验证

- 输入：代码改动与验收标准
- 输出：测试报告
- 门槛：核心场景验证通过，风险点已记录

### 8. 提交与 PR

- 输入：代码改动、测试报告、关联文档
- 输出：规范化 commit 与 PR
- 门槛：提交信息、PR 描述与文档链接完整

### 9. 评审合并

- 输入：PR、测试证明、评审意见
- 输出：合并结果与必要修正
- 门槛：评审通过，质量门禁通过

### 10. 文档回写与复盘

- 输入：已合并改动
- 输出：成员状态更新、模块文档补全、复盘总结
- 门槛：文档与代码状态一致
