# daily-report 设计文档

## 1. 目标

为团队成员提供统一的个人日报生成 Skill，自动读取成员状态和任务视图，生成当日日报草稿，并保留阻塞项与明日计划的补充位。

## 2. 边界

`daily-report` 只负责个人日报生成，不替代任务池管理或状态同步。

它负责：

- 读取成员 `status.md`
- 读取成员 `tasks.md`
- 生成日报草稿文件
- 保留阻塞项和明日计划的补充位

它不负责：

- 修改任务池状态
- 发布或认领任务
- 生成周报、月报等其他报告

## 3. 输入

- 成员名
- 成员目录路径
- 日期
- 可选的手工补充说明

## 4. 输出

- `docs/成员名/reports/daily/YYYY/YYYY-MM-DD.md`

## 5. 依赖模板

- `templates/member/daily-report.md`

## 6. 生成逻辑

- 从 `status.md` 读取当前主任务和阻塞信息
- 从 `tasks.md` 读取各状态列表
- 组合为日报草稿
- 留下“阻塞项”和“明日计划”的补充位置
