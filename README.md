# agri-dev-governance

智慧农业团队的组织级工程治理仓库，用于统一多项目开发规范、Skill 工作流、文档模板、Git/GitHub 协作规则与质量门禁，确保团队成员与 AI 助手都遵循同一套可追溯、可执行、可校验的研发流程。

## 仓库定位

本仓库不承载业务代码，专门负责沉淀和维护团队研发治理底座，覆盖物联网、数据中台、业务系统以及未来新增项目的协作规范。

## 核心原则

- 先规范，后开发
- 先文档，后实现
- 先设计，后编码
- 先计划，后执行
- 先验证，后提交
- 所有开发过程必须可追溯

## 治理范围

- 团队开发规范与工程手册
- 成员初始化与任务状态管理
- 需求、设计、实施、测试等文档模板
- Git、worktree、commit、PR、review 规则
- 本地脚本与 CI 质量门禁
- AI 与人工协作的统一工作流

## 推荐使用顺序

1. 阅读团队工程手册
2. 执行成员初始化
3. 按模块建立文档目录
4. 先完成需求与设计文档
5. 产出实施计划后再进入开发
6. 开发完成后补齐测试报告并按规范提交

## 目录说明

- `docs/overview/`：治理总览、工程手册、流程总览
- `docs/workflows/`：各流程规范正文
- `docs/visuals/`：可视化流程与前端资源
- `docs/training/`：培训材料与使用手册
- `docs/records/`：设计文档与实施计划归档
- `skills/`：团队自定义 Skill
- `templates/`：统一文档模板
- `scripts/`：本地与 CI 校验脚本
- `examples/`：示例产物与参考结构
- `.github/`：Issue / PR / Actions 模板
